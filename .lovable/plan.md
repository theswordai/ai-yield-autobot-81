## 全站公共聊天大厅 — 实施计划

### 一、入口位置
在右下角悬浮按钮区，与现有「在线客服」（`MessageCenter`）并列新增一个「聊天大厅」气泡按钮。

```text
              ┌──────────┐
              │ 💬 客服  │  ← 已有
              ├──────────┤
              │ 👥 大厅  │  ← 新增（更靠下，间隔 14px）
              └──────────┘
```

- 移动端：`bottom-24 right-4`（客服）→ 大厅放 `bottom-40 right-4`
- 桌面端：`bottom-6 right-6`（客服）→ 大厅放 `bottom-24 right-6`
- 同样使用 glass-morphism 样式（与全站设计 token 一致）
- 未读消息数小红点（最近 N 条新消息中未被本机已读的数量）

点开后弹出右侧 `Sheet` 面板（与客服一致的交互），内为聊天大厅。

### 二、功能与发言权限

**所有人**（含未连接钱包）：可阅读全站公共消息流。
**可发言条件**（满足其一即可，前端 + 数据库双校验）：
1. 该钱包在 `referrals`/`inviter_bindings` 等已绑定邀请人；或
2. 该钱包在质押合约中持有有效质押头寸（通过现有 `useStakingData` 链上读取判定）。

未满足条件时输入框禁用，并显示提示：
> 「需绑定邀请人或拥有有效质押才能发言。前往「投资」页参与质押或绑定邀请人。」

### 三、消息体验
- 时间倒序加载最近 200 条，向上滚动加载更早。
- 实时订阅 Supabase Realtime，新消息自动追加。
- 每条消息显示：截短钱包地址（0x12..ab）、内容、时间、可选的 VIP/管理员小徽章。
- 管理员可删除任意消息（复用 `useIsAdmin`）。
- 单条消息长度上限 500 字，前端 + 触发器限制。
- 简单防刷：每个钱包 3 秒一条（前端节流 + 表索引时间校验）。
- 不支持图片/文件（先做纯文本，保持轻量）。

### 四、数据库
新建一张表 `chat_messages`，并加入 Realtime 发布。

字段（业务相关）：
- `wallet_address`：发言人钱包
- `content`：内容（≤500 字符）
- `is_deleted`：软删除标记

RLS（沿用项目既有"无 auth.uid 的客户端写入"模式）：
- 任何人可读未删除消息
- 任何人可插入（前端校验 + 触发器再校验长度/频率）
- 任何人可更新 `is_deleted`（管理操作前端限制为管理员）
